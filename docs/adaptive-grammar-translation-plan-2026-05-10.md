# 適応型文法・和訳練習の実装方針

## 方針

MedAce の CBT は、公式 CEFR 判定や本格 CAT 試験ではなく、学習用の adaptive practice として扱う。校正済み item bank、能力推定の信頼性、停止規則、公平性の証拠が揃うまでは、「現在の練習範囲での目安」として表示する。

## 採用する学習設計

- `basic-svo` を「3ます: 主語 / 動詞 / 目的語」の中核練習にする。
- 文法範囲ごとに、型、受験で狙われる点、よくあるミス、自動化ドリルを持たせる。
- 和訳全文入力は完全一致だけでなく、意味、文法構造、受験答案らしさでフィードバックする。
- AI 生成問題は難しい英文ではなく、既習語と高頻度語中心の i+1 問題を優先する。
- 復習設計は、検索練習と間隔反復を前提に、知る、できる、自動化の順で支援する。

## CBT の扱い

- 生成問題は `ai_generated_problems` に保存し、再利用して API 料金を抑える。
- `assessment_item_metadata` は校正状態、レビュー状態、construct、item format を持つ。
- 学習者状態は全体 ability だけでなく、`cbt_learner_scope_states` で文法範囲と問題形式ごとに更新する。
- 少数データ時は IRT 確定値ではなく、Elo 風の bounded update と正答率ベースの暫定難度に留める。

## 避けること

- AI 生成直後の問題を「校正済み」と表示しない。
- 文法穴埋めや並び替えだけで総合英語力を断定しない。
- CEFR/ACTFL の公式 rating に見える UI を出さない。
- 入力時間を強い能力指標にしない。
