# Project: Steady Study

## 1. Product Positioning

Steady Study は、塾・教室向け運用SaaSをメインにした英単語学習プロダクトです。個人学習アプリはサブ導線として残しますが、事業の中心は「講師が多人数を運用しやすくすること」に置きます。

- メイン: 塾・教室向け運用SaaS
- サブ: 個人学習アプリ
- ブランド名: `Steady Study`
- インフラ前提: Cloudflare Pages + Pages Functions + D1

## 2. 顧客課題

### 塾・教室

- 生徒の学習停止を講師が見逃しやすい
- 誰がどの生徒を見るかが曖昧になりやすい
- 教材権利と公開範囲を運用で切り分けにくい
- 個別学習プランを講師の属人的判断に頼りがち

### 個人学習

- 最初に何をどれだけやるか決めにくい
- 自分に合った教材と復習ペースが見えにくい

## 3. 提供価値

### 生徒向け

- 静的12問の初回診断で開始レベルをすぐ確定
- SRSベースで今日やる復習と新規学習を自動提示
- 試験日、目標点、学習時間、苦手領域を前提に学習プランを生成

### 講師向け

- 生徒ごとのリスク理由と次アクションを一覧化
- 担当講師の割当を前提に閲覧範囲を制御
- 通知文のAI下書きを使いながら、講師名義で自然な再開フォローを送信

### グループ管理者向け

- 組織全体の割当率、未割当生徒数、学習プラン浸透率を可視化
- 講師ごとの担当人数と通知数を一画面で確認
- 教材カタログを権利区分ごとに配布

## 4. 収益モデル

- 固定費: 10,000円 / 月
- 生徒課金: 2,000円 / 人 / 月
- 講師課金: 500円 / 人 / 月
- 導入費: 600,000円

無料トライアル (`TOB_FREE`) は導入前検証用、本導入 (`TOB_PAID`) で正式な教材権限と運用機能を提供します。

## 5. KPI

- 通知後の学習再開率
- 学習プラン浸透率
- 担当講師割当率
- 7日継続率
- 講師1人あたりの運用生徒数

## 6. 教材戦略

- `STEADY_STUDY_ORIGINAL`
  - 出典: Nanjyo English App のオリジナル単語データベース
  - 公開範囲: `BUSINESS_ONLY`
  - 用途: ビジネス本導入向けの原本教材
- `LICENSED_PARTNER`
  - 出典: 現在のライセンス取得済み教材データベース
  - 公開範囲: `BUSINESS_ONLY`
  - 用途: ビジネス本導入向けの正式教材
- `USER_GENERATED`
  - 出典: ユーザー/管理者作成
  - 公開範囲: 原則 `ALL_PLANS`

2026-03-06 時点の方針として、既存の公式教材は一括で `BUSINESS_ONLY` に寄せ、個人/無料ユーザーの可視性を大きく絞る。

## 7. 技術構成

- Frontend: React 19 / Vite / Tailwind CSS / Lucide React
- Backend: Cloudflare Pages Functions
- Database: Cloudflare D1
- Offline fallback: IndexedDB
- AI: Gemini API をサーバー経由で利用

初回診断はAI生成ではなく、事前定義した12問バンクを使います。AIは診断後の学習支援に限定します。

## 8. データモデル

- `users`
- `books`
- `words`
- `learning_histories`
- `learning_plans`
- `learning_preferences`
- `instructor_notifications`
- `student_instructor_assignments`
- `ai_usage_events`

## 9. 権限の考え方

- システム管理者: 全体管理
- グループ管理者: 自組織全体を閲覧・割当変更
- 講師: 担当生徒と未割当生徒を中心に閲覧・通知
- 生徒: 自分の学習データのみ閲覧

この段階では講師に未割当生徒も見せる運用ですが、今後はクラス単位権限や担当履歴の導入でさらに絞り込みます。
